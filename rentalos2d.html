Guía de configuración Firebase
Paso a paso para no-expertos en backend · ~20 minutos
1
Firebase Console
Crear proyecto Firebase
Ve a console.firebase.google.com e inicia sesión con tu cuenta Google.
Clic en "Agregar proyecto" → ponle nombre (ej. rentalos-prod) → desactiva Google Analytics si no lo necesitas → "Crear proyecto".
2
Web App
Registrar tu app web y obtener firebaseConfig
En la consola del proyecto → ícono "</>" (Web) → registra la app con nombre (ej. rentalos-web).
Firebase te mostrará un bloque firebaseConfig. Cópialo completo. Luego en index.html busca la sección marcada con 🔧 y pega tus valores reales:
const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "rentalos-prod.firebaseapp.com", projectId: "rentalos-prod", storageBucket: "rentalos-prod.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" };
⚠️ Importante: El apiKey de Firebase para web es público por diseño — no lo escondas en variables de entorno. La seguridad real se configura con Firestore Rules y Auth (pasos siguientes).
3
Authentication
Activar Phone Authentication (SMS)
En la consola → Build → Authentication → Get started
En la pestaña "Sign-in method" → activa "Phone" → guarda.

Configurar dominios autorizados: en la misma sección de Authentication → pestaña "Settings" → "Authorized domains" → agrega tu dominio de producción (ej. rentalos.com). localhost ya viene incluido para desarrollo.
✓ Para pruebas locales: Puedes agregar números de prueba en Authentication → Sign-in method → Phone → "Phone numbers for testing". Ej: +52 449 000 0001 con código 123456. Así evitas cargos de SMS durante desarrollo.
4
reCAPTCHA
reCAPTCHA — sin configuración extra necesaria
Firebase Phone Auth usa reCAPTCHA automáticamente. El código ya lo configura con RecaptchaVerifier.
En producción, debes agregar tu dominio en Google reCAPTCHA Admin Console (search.google.com/recaptcha/admin) → tu sitio tiene que estar listado. Firebase ya incluye su propio dominio, pero el tuyo propio necesita registro si usas reCAPTCHA enterprise.
Nota: En localhost el reCAPTCHA aparece visible. En producción con dominio verificado, Firebase puede hacer la verificación invisible automáticamente.
5
Firestore
Activar Firestore Database
En la consola → Build → Firestore Database → Create database
Selecciona "Start in production mode" (más seguro) → elige la región más cercana a México: us-central1 o nam5 → Listo.
Estructura de colecciones que usa Rentalos:
Firestore ├── users/ │ └── {uid}/ ← documento por usuario │ ├── uid │ ├── name │ ├── phone │ ├── city │ ├── colonia │ ├── bio │ ├── schedule │ ├── createdAt │ └── updatedAt │ └── listings/ └── {auto-id}/ ← documento por publicación ├── uid ← UID del dueño ├── ownerName ├── ownerPhone ├── title ├── cat ├── desc ├── cond ├── avail ├── zone ├── city ├── emoji ├── createdAt └── updatedAt
6
Seguridad
Configurar Firestore Security Rules
En la consola → Firestore → pestaña "Rules" → pega estas reglas y publica:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // ── USUARIOS ────────────────────────────── match /users/{uid} { // Cualquiera puede leer perfiles (para mostrar nombre en cards) allow read: if true; // Solo el propio usuario puede crear/editar su perfil allow create: if request.auth != null && request.auth.uid == uid && request.resource.data.uid == uid && request.resource.data.phone is string && request.resource.data.name is string && request.resource.data.name.size() > 0 && request.resource.data.name.size() <= 100; allow update: if request.auth != null && request.auth.uid == uid && request.resource.data.uid == uid; allow delete: if false; // No permitir borrar perfiles } // ── PUBLICACIONES ───────────────────────── match /listings/{listingId} { // Cualquiera puede leer publicaciones allow read: if true; // Solo usuarios autenticados pueden publicar allow create: if request.auth != null && request.resource.data.uid == request.auth.uid && request.resource.data.title is string && request.resource.data.title.size() > 0 && request.resource.data.title.size() <= 200 && request.resource.data.cat in [ 'herramientas','servicios','hogar','jardin', 'electrodomesticos','deporte','tecnologia', 'vehiculos','otro' ]; // Solo el dueño puede editar su publicación allow update: if request.auth != null && resource.data.uid == request.auth.uid && request.resource.data.uid == request.auth.uid; // Solo el dueño puede eliminar su publicación allow delete: if request.auth != null && resource.data.uid == request.auth.uid; } } }
✓ Estas reglas garantizan: lectura pública, escritura solo autenticada, edición/borrado solo del dueño, validación de campos requeridos, y lista blanca de categorías.
7
Probar localmente
Probar Phone Auth en localhost
Abre el archivo index.html directamente en el navegador con Live Server (extensión de VS Code) o desde un servidor local. NO abras el archivo directamente con file:// — Firebase requiere un servidor HTTP aunque sea local.
Opciones rápidas:
# Opción 1 — Python (ya instalado en Mac/Linux) python3 -m http.server 8080 # Luego abre: http://localhost:8080 # Opción 2 — Node.js npx serve . # Luego abre la URL que muestre # Opción 3 — VS Code Live Server # Click derecho en index.html → "Open with Live Server"
✓ Números de prueba: En Firebase Auth → Sign-in method → Phone, agrega un número ficticio (ej. +52 449 000 0001) con un código fijo (ej. 123456). Así puedes probar sin gastar SMS reales.
8
Despliegue
Publicar en producción
Opción A — Firebase Hosting (recomendada, gratis):
# 1. Instalar Firebase CLI npm install -g firebase-tools # 2. Login firebase login # 3. En la carpeta del proyecto firebase init hosting # → Selecciona tu proyecto # → Public directory: . (punto, el directorio actual) # → Single-page app: No # → Overwrite index.html: No # 4. Desplegar firebase deploy
Opción B — Netlify (drag and drop):
Ve a app.netlify.com → "Add new site" → "Deploy manually" → arrastra tu carpeta. Listo. Te da HTTPS automático.
Opción C — GitHub Pages:
Sube el archivo a un repositorio público en GitHub → Settings → Pages → Deploy from branch (main, / root).
9
Costos y límites
Costos Firebase (plan Spark — gratuito)
El plan Spark de Firebase es gratuito e incluye:
Firestore: 1 GB almacenamiento, 50k lecturas/día, 20k escrituras/día Auth: 10,000 verificaciones por teléfono/mes (gratis) Hosting: 10 GB/mes transferencia, 1 GB almacenamiento Functions: No incluido en este proyecto
⚠️ Anti-abuso de SMS: Firebase limita automáticamente los SMS. Para proyectos en producción agrega una capa extra: activa "App Check" en Firebase console y configura un presupuesto de alerta en Google Cloud Console para recibir notificación si el uso dispara.
10
Errores comunes
Troubleshooting — errores más frecuentes
Error: auth/invalid-phone-number → El número debe incluir código de país: +52 449 123 4567 (no espacios ni guiones en el envío, el código los limpia) Error: auth/too-many-requests → Firebase bloqueó temporalmente por intentos repetidos. Espera 1-2 minutos o usa números de prueba para desarrollo. Error: auth/captcha-check-failed → reCAPTCHA no pasó. Asegúrate de abrir el sitio en un servidor HTTP (no file://). Prueba en modo incógnito. Error: Missing or insufficient permissions → Las Firestore Rules están bloqueando. Verifica que el usuario esté autenticado y que publicaste las rules. reCAPTCHA no aparece en el modal → El widget se monta cuando abres el modal por primera vez. Si se monta dos veces, refresca la página. Listados no cargan en producción → Verifica que el dominio esté en Authentication → Authorized domains en la consola de Firebase.

Próximos pasos
Recomendaciones para escalar Rentalos
Corto plazo:
• Firebase Storage para fotos reales de los objetos
• Firestore index compuesto (cat + createdAt) para filtros rápidos
• Firebase App Check para prevenir abuso de la API

Mediano plazo:
• Cloud Functions para enviar notificaciones o moderar contenido
• Algolia o Typesense para búsqueda full-text avanzada
• Sistema de reputación/calificaciones entre vecinos

Largo plazo:
• App móvil con React Native + Firebase (mismo backend)
• Expansión a más ciudades con multi-tenant por zona
• Integración con WhatsApp Business API
¿Dudas? El código ya tiene todo configurado — solo necesitas pegar tu firebaseConfig real.
Tiempo estimado de setup: 15–25 minutos desde cero.
